3

취약점 실습 공격

DVWA Security Level을 Low로 설정 후 SQL Injection과 XSS 공격을 단계별로 수행했습니다.

① SQL Injection — DB 구조 파악 후 데이터 탈취

# 1단계: 취약점 확인 1' → SQL 에러 발생 확인 # 2단계: 컬럼 수 파악 1' ORDER BY 2# → 정상 / 1' ORDER BY 3# → 에러 → 컬럼 2개 # 3단계: DB 이름 파악 1' UNION SELECT database(), null# → dvwa # 4단계: 테이블 목록 1' UNION SELECT table_name, null FROM information_schema.tables WHERE table_schema='dvwa'# # 5단계: 컬럼 목록 1' UNION SELECT column_name, null FROM information_schema.columns WHERE table_name='users'# # 6단계: 데이터 추출 1' UNION SELECT user, password FROM users# → 계정·해시 노출
🖼SQLi 공격 성공 — users 테이블 계정·해시 노출 화면

④ SQL Injection — 계정·패스워드 해시 탈취 성공

② XSS (Reflected)

# 입력창에 스크립트 삽입 <script>alert('XSS')</script> # 결과: alert 팝업 실행 → 스크립트 인젝션 가능 확인
🖼XSS 공격 성공 — alert 팝업 화면

⑤ XSS Reflected — 스크립트 실행 확인

취약점 조치 (After)

소스 코드 수정 및 Apache 설정 변경으로 각 취약점을 조치하고 재스캔으로 결과를 검증했습니다.

① SQL Injection — Prepared Statement 적용

BEFORE — 취약한 코드
$query = "SELECT ... WHERE user_id = '$id'";
AFTER — Prepared Statement
$stmt = $pdo->prepare( "SELECT ... WHERE user_id = ?"); $stmt->execute([$id]);
🖼조치 전 — 공격 성공

조치 전 — 데이터 탈취 성공

🖼조치 후 — 공격 실패

조치 후 — 결과 없음 / 에러

② XSS — 입력값 이스케이프 처리

BEFORE
echo "Hello " . $_GET['name'];
AFTER
echo "Hello " . htmlspecialchars( $_GET['name'], ENT_QUOTES);

③ Apache 버전 정보 노출 조치

BEFORE — curl -I 응답
Server: Apache/2.4.52 (Ubuntu) X-Powered-By: PHP/8.1.2
AFTER — 버전 숨김
Server: Apache
🖼security.conf vi 편집 화면 / curl -I Before·After 비교

⑥ ServerTokens Prod / ServerSignature Off 설정 변경

🖼Nikto 재스캔 결과 — 취약점 감소 확인

⑦ Nikto 재스캔 (After) — 조치 후 취약점 항목 감소

1

2
Logstash 파이프라인 설정
/etc/logstash/conf.d/apache.conf 작성 — input(file) → filter(grok) → output(elasticsearch)
3
Kibana 대시보드 설정
Index Pattern 생성, 시각화 패널 구성
4
서비스 상태 확인
systemctl status elasticsearch / logstash / kibana — 모두 active 확인
# Logstash 파이프라인 설정 (apache.conf) input { file { path => "/var/log/apache2/access.log" start_position => "beginning" } } filter { grok { match => { "message" => "%{COMBINEDAPACHELOG}" } } } output { elasticsearch { hosts => ["localhost:9200"] index => "apache-logs-%{+YYYY.MM.dd}" } }
