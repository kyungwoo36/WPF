GNS3 기반 네트워크 보안 환경 구축

GNS3와 VirtualBox를 활용해 실제 기업 네트워크와 유사한 가상 보안 환경을 구성했습니다. 방화벽 룰 설계, OpenVPN 구축, Snort IDS 룰셋 작성, OSSEC HIDS 연동까지 네트워크 보안의 전 과정을 직접 구현하고 공격 탐지를 검증했습니다.

도구
GNS3 · pfSense · Snort · OSSEC
환경
VirtualBox · Kali Linux · CentOS 7 · Win10
유형
홈랩 실습 프로젝트
1

네트워크 토폴로지 설계

GNS3 토폴로지 — 내부망 / DMZ / 외부망 3구간 구성

내부망·DMZ·외부망 3구간으로 분리된 네트워크를 설계하고, pfSense 방화벽으로 구간 간 트래픽을 제어했습니다. OSSEC Agent 위치, VPN 게이트웨이, 웹서버 구간의 방어 전략이 포함됩니다.

  • R
    Firefox3(내부) → HTTP · Telnet 허용
  • R
    Webterm2(DMZ) → Telnet 허용
  • R
    PC1 → Webterm1 Ping 허용
  • R
    Webterm1 ↔ R1 HTTP · Telnet 허용
  • R
    Webterm2 → PC1 Ping 허용

OpenVPN 구축

1
CA 및 서버 인증서 생성
pfSense 웹 콘솔 → OpenVPN Wizard에서 CA와 서버 인증서 생성
2
OpenVPN 서버 설정
터널 네트워크, 로컬 네트워크 대역 설정 및 암호화 알고리즘 지정
3
사용자 계정 및 방화벽 룰 설정
System → User Manager에서 VPN 계정 생성, OpenVPN 인터페이스 방화벽 룰 허용
4
클라이언트 연결 테스트
Client Export에서 Ubuntu용 .ovpn 파일 다운로드 후 사용자 ID/PW 입력해 접속 확인
3

Snort IDS 룰셋

Snort 룰 적용 화면

# HTTP 접근 탐지
alert tcp any any -> $HOME_NET 80 (msg:"HTTP Access Detected"; sid:1000001;)

# DDoS 랜덤 소스 탐지
alert tcp any any -> $HOME_NET any (flags:S; threshold:type both,track by_dst,count 100,seconds 1; msg:"Possible DDoS"; sid:1000002;)

# SYN 스캔 탐지
alert tcp any any -> $HOME_NET any (flags:S; msg:"SYN Scan Detected"; sid:1000003;)
